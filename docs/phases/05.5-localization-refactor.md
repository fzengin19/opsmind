# Refactor: Native Carbon Localization

## Goal
Remove manual arrays for day names (`days_short`, `days_full`) from translation files and replace all usages with Carbon's native `translatedFormat()` or `isoFormat()` methods. This ensures consistency, reduces maintenance, and leverages the framework's built-in localization power.

## 1. Clean Up Language Files
**Files:** `lang/tr/calendar.php`, `lang/en/calendar.php`
- Remove `days_short` array.
- Remove `days_full` array.
- Remove `months` array (if exists - verified it doesn't).

## 2. Update Views & Logic
**Location:** `resources/views/livewire/calendar/index.blade.php`
- **Current:** Loop uses `$dayKey` (sun, mon...) to look up translation.
- **New:** Use the Carbon date object directly.
    - Replace reference to `day['name']` or translation key with `$day['date']->translatedFormat('D')` (short day) or `l` (full day).

**Location:** `app/Services/CalendarService.php`
- **Method `getMonthGrid`**: Ensure it returns Carbon objects or properly localized strings.
    - Check usage of `dayName`.
    - Replace `$current->dayName` (which might rely on locale if set manually) with `$current->translatedFormat('D')` logic if meant for display. But actually, `dayName` property on Carbon *is* localized if locale is set.
    - **Optimization:** We just stripped `locale('tr')` calls. Now we must rely on the App locale.

**Location:** `resources/views/livewire/calendar/day-detail.blade.php`
- Verify no other manual lookups exist (mostly cleaned already).

## 3. Implementation Steps
1.  **Modify `index.blade.php`:** Update the table header loop (Mon, Tue...) to generate day names dynamically from a reference week date.
2.  **Modify `CalendarService.php`:** Ensure `dayName` in grid generation is either removed (if unused) or generated using `translatedFormat`.
3.  **Clean `lang` files:** Delete the arrays.
4.  **Verify:** Check the calendar headers match the active app locale.
